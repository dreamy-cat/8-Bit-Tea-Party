# Tools and utilities for all our retro platforms

 External tools for games and demos development by our team, target retro platforms and/or just text or data files of platform's specific formats. Use classic C programming language C11+ and CMake, verified only on GCC. ;) Works with data, texts, images, music, etc.

# Инструментарий разработки для всех наших ретро-платформ

 Внешний инструментарий для разработки игр и демок нашей командой. Результат используется как напрямую ретро платформами, так и просто текстовыми или файлами данных их формата. Применяем классический Си, стандарт от 11-ого и выше и немного конфигураций, протестированных пока только с GCC. ;) Утилиты для обработки данных, текстов, графики и музыки и т.д.

## Общие задачи и комментарии

 В целом пока что это черновик, но частично уже можно использовать. Привести всё в порядок и самое главное, добавить главное меню и возможно выравнивания для консоли при выводе. Спрайты с плюсов перетянуть на Си. Остальные задачи можно посмотреть напрямую в самих файлах, постепенно по мере необходимости менять структуру инструментов. Не забывать про подробный вывод во время действия, чтобы быстро вносить изменения и понимать что происходит. Комментарии добавить по желанию в исходники, возможно задуматься о языках интерфейса. Еще возможно стоит загрузку и выгрузку из/в текствоый файл сделать как отдельный функционал с гибкими настройками.
 
## Список утилит и их краткое описание

 1. Утилита для формирования файла палитры цветов графического экрана или спрайта из стандартного BMP файла. Поддержка платформ и адаптеров, плюс наши форматы для VGA-X, упакованы или не очень. Загрузка и сохранение в графический или текствоый файл уже работают.
 2. Преобразование стандартного монохромного спрайта в виде BMP файла, в данные для спрайта нашего движка при отрисовке анимации на экране. Атрибуты не учитываются, возможно в будущих версиях. Формат слева-направо, сверху-вниз и с выравниванием по размеру байта, а не слова или двойного слова. Обрабатывает все файлы в текущей директории, удобней пока собирать статичную сборку.

## Краткое описание файлов

 - CMakeLists.txt - файл проекта или можно напрямую использовать исходники;
 - main.c - основной файл всех утилит, пока что в нем только прямые вызовы, но переделать под меню и интерфейс только через систему меню и клавиатуры, имеет-ли смысл реализовать поддержку командной строки вопрос открытый;
 - palettes.c - обработка палитр из файлов BMP, черновик, но уже базовый функционал у нас работает и можно применять для VGA-X режима и экпорта в текст;
 - sprites.c - быстрый файл для изготовления спрайта из bmp файла для нашего движка ZX Spectrum, просто данные, которые можно уже дальше переносить как .C файл.
 - readme.md - этот файл с общим описанием.
